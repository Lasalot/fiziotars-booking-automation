booked4.us REST API v2 Endpoints
This document summarises the endpoints available in the booked4.us REST API v2. It groups them by resource tags and explains the purpose of each endpoint along with usage details.
Overview
The booked4.us API enables programmatic access to calendar management, reservations, user management, service definitions and other resources. Endpoints typically return JSON objects and require authorization via OAuth2. The API uses standard HTTP methods to create (POST), read (GET), update (PUT), and delete (DELETE) resources.
